# Аналіз ринку та аналогів: 2-осьовий підвіс (Pitch/Roll) для мультиспектральної камери БПЛА (варіант 4)

## 1) Контекст і мета аналізу
Мета цього розділу — порівняти наявні на ринку рішення для стабілізації камер на БПЛА та обґрунтувати доцільність кастомного 2-осьового підвісу під мультиспектральний сенсор (MicaSense RedEdge‑P), з керуванням через S.Bus/UART і живленням через DC‑DC/BEC.

Фокус аналізу:
- 2 осі стабілізації (Pitch/Roll), утримання надиру, мінімізація паразитних моментів від кабелів.
- Сумісність з мультиспектральними камерами (маса/габарити/центр мас) і наявність Trigger/Sync.
- Реальна інтеграція на платформу (живлення, кабель‑менеджмент, монтаж, демпфери), а не тільки “номінальні” характеристики.

## 2) Критерії оцінки аналогів (єдина шкала порівняння)
Рекомендовано збирати дані по кожному аналогу за цими параметрами (мінімум):
1. **Кількість осей**: 2 / 3.
2. **Підтримка камер**: типи сенсорів, максимальна маса, геометрія кріплення.
3. **Інтерфейси керування**: S.Bus / PWM / UART / CAN / SDK / інше.
4. **Живлення**: діапазон Vin, наявність/відсутність окремого DC‑DC, доступні Vout.
5. **Стабілізація/точність**: заявлена точність, частота контурів, наявність IMU/калібрувань (якщо є).
6. **Механіка**: прямий привід BLDC / редуктор; демпфери; кабель‑канали; ремонтопридатність.
7. **Маса підвісу** (без камери) та **габарити**.
8. **Вартість і доступність**: орієнтовна ціна, доступність в Україні/ЄС, строки.
9. **Обмеження інтеграції**: екосистема виробника, “закриті” протоколи, прив’язка до платформи/FC.

> Важливо: цифри (маса, ціна, діапазон живлення) вносити тільки зі специфікацій/офіційних сторінок. У таблиці нижче залишені поля для заповнення.

## 4) Типові висновки з ринку (попередня рамка)
Після заповнення таблиці сформулюй 3–6 тез, що прямо ведуть до рішення “кастомний підвіс”:

**4.1. Екосистемна залежність**
Багато “преміум” рішень є частиною закритих екосистем (SDK/FC/камери), що ускладнює інтеграцію мультиспектрального сенсора або обмежує доступ до Trigger/Sync та телеметрії.

**4.2. Вартість проти гнучкості**
Готові промислові підвіси забезпечують стабілізацію “з коробки”, але мають суттєво вищу ціну та часто не оптимізовані під конкретну масу/центр мас камери, через що виникають зайві витрати по енергії та масі.

**4.3. Питання живлення і кабель‑менеджменту**
На практиці ключові проблеми — це не “2 чи 3 осі”, а: коректне живлення (Vin → стабільний Vout), розведення кабелів без паразитних моментів, демпфери/вібрації та правильне розташування IMU/центру мас.

**4.4. Реальна ремонтопридатність**
Кастомна конструкція (площадки, демпфери, кронштейни) під конкретну платформу дає швидку заміну витратників і компонентів, а також можливість локальної модернізації.

## 5) Висновок щодо позиціювання обраної моделі (варіант 4)
Запропонований 2-осьовий підвіс під мультиспектральну камеру позиціюється як:
- **цільове рішення** під конкретний сенсор і політний профіль (60–100 м, надир);
- **відкрита інтеграція** з FC через **S.Bus/UART** і штатний **Trigger/Sync**;
- **проста підтримка**: окремий DC‑DC/BEC, контрольовані точки кріплення, сервісна доступність.

## 6) Ризики та способи зниження (для завершеності звіту)
1. **Недостатній момент моторів** → розрахунок по масі/плечу + запас 30–50% по моменту; тест на стенді.
2. **Вібрації/резонанси** → демпфери, балансування, жорсткість рам, правильне прокладання кабелів.
3. **Нестабільне живлення** → DC‑DC з запасом по струму, фільтрація, окремі лінії живлення для камери/контролера.
4. **Сумісність інтерфейсів** → перевірити конкретну модель контролера підвісу на підтримку S.Bus/UART і режимів керування.

## 7) Що потрібно додати перед здачею
- Заповнити таблицю аналогів конкретними моделями та **посиланнями**.
- Внести точні значення по живленню/масі/інтерфейсам з джерел.
- Додати 3–6 фінальних тез висновків (на основі заповненої таблиці).
- За потреби — короткий абзац про “чому 2 осі достатньо” для надиру (з фокусом на масу/енергоспоживання).
